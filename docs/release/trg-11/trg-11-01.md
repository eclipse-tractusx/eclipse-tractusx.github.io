## Why

Generative AI (GenAI) tools — such as GitHub Copilot — are increasingly used to assist software development. To keep Eclipse Tractus-X a clean and redistributable open source project, the usage of these tools must respect the FOSS governance rules defined by the Eclipse Foundation.

The source of truth is always the [Eclipse Foundation Project Handbook — Generative AI](https://www.eclipse.org/projects/handbook/#genai).

This TRG describes **how** contributors may use Generative AI when contributing to Eclipse Tractus-X. The legal disclosure and licensing obligations for the produced content are described in [TRG 11.02 - AI disclosure and licensing in headers](/docs/release/trg-11/trg-11-02).

:::info

The requirements described here **must** be met for each contribution that was created with the assistance of Generative AI.

:::

## Description

Generative AI tools may be used to contribute to Eclipse Tractus-X, provided that:

- The contributor has the **right to contribute** the generated content under the project's license.
- The contributor has **reviewed and verified** that the generated content is correct, of good quality, and free of obvious license or security issues.
- The contribution is **disclosed** as described below and in [TRG 11.02](/docs/release/trg-11/trg-11-02).

:::tip

GitHub Copilot is available for all Eclipse committers who request it, free of charge. It may be used for Eclipse Tractus-X contributions as long as the disclosure and licensing rules of [TRG 11.02](/docs/release/trg-11/trg-11-02) are followed.

:::

## Disclosing AI assistance in commits

When a commit contains content that was generated with the assistance of Generative AI, the commit message **must** include an `Assisted-by:` trailer that names the tool **and** the concrete model (including its version) that was used.

```text
Hide unwanted action bar items

This change hides unwanted 'Link with Editor' and
'Customize View...' items from the local toolbar
and the view menu.

Assisted-by: GitHub Copilot (GPT-4o-2024-08-06)
```

**Notes:**

- The trailer goes at the **end** of the commit message, together with any other trailers (e.g. `Signed-off-by:`).
- Always record the **specific model and version** that produced the contribution (for example `GitHub Copilot (Claude-Opus-4.8-2026-06-16)` or `Google (gemini-2.5-pro)`), not just the vendor name.
- One `Assisted-by:` trailer is added per tool/model that contributed to the commit.

## AI usage policy (`AI_POLICY.md`)

Each repository **should** provide an `AI_POLICY.md` file at its root that makes the project's stance on Generative AI transparent to every contributor.

The `AI_POLICY.md` file should clarify, at minimum:

- Whether the usage of Generative AI is **allowed, restricted, or forbidden** in the repository.
- Which **models or tools** are recommended or permitted.
- The **quality and verification requirements** a contribution has to fulfil (for example: human review, accompanying tests, no low-quality or unverified output).
- A reference to the disclosure and licensing obligations in [TRG 11.02](/docs/release/trg-11/trg-11-02).

This keeps the rules transparent and consistent for every contributor, both in repositories that allow Generative AI and in those that restrict it.
