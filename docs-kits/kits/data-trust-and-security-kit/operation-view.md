## Operation view

## Requirements for Identifiers

- The DID Document requires to be available for the public internet and include the public key.
- Additionally other identifiers can be available in the DID document.

## Requirements for Semantic JSON-LD @contexts and Verifiable Credentials

- The JSON-LD @contexts must be hosted in a publicly accesible endpoint, so they can be accesible by any data consumer.
- The verifiable credentials must be valid JSON-LDs and for validating that, [JSON-LD Playgrounds](https://json-ld.org/playground/) can be used for manual verification or an standard JSON-LD library like [pyld](https://github.com/digitalbazaar/pyld) and others for automatic semantic verification.

## Governance Body Requirements

- The published rulebooks require to be publicly accessible.
