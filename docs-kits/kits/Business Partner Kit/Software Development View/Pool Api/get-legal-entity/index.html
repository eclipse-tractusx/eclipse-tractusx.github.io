  • Returns a legal entity by identifier, like BPN, DUNS or EU VAT ID, specified by the identifier type
Version: 24.05

Returns a legal entity by identifier, like BPN, DUNS or EU VAT ID, specified by the identifier typeâ€‹

This endpoint tries to find a business partner by the specified identifier. The identifier value is case insensitively compared but needs to be given exactly. By default the value given is interpreted as a BPN. By specifying the technical key of another identifier typethe value is matched against the identifiers of that given type.

Path Parameters
  • idValue string required

    Identifier value

Query Parameters
  • idType string

    Default value: BPN

    Type of identifier to use, defaults to BPN when omitted

Responses
  • 200
  • 400
  • 404

Found business partner with specified identifier

  • application/json
  • Schema
  • Example (from schema)
Schema
  • bpnl string

    A BPNL represents and uniquely identifies a legal entity, which is defined by its legal name (including legal form, if registered), legal address and tax number.

  • legalName string

    The name of the legal entity according to official registers.

  • legalShortName string

    The abbreviated name of the legal entity.

  • legalForm object

    A legal form is a mandatory corporate legal framework by which companies can conduct business, charitable or other permissible activities.

  • technicalKey string

    The technical identifier of the legal form according to ISO 20275.

  • name string

    The name of legal form according to ISO 20275.

  • abbreviation string

    The abbreviated name of the legal form, such as AG for German Aktiengesellschaft.

  • identifiers object[]

    The list of identifiers of the legal entity.

  • value string

    The value of the identifier like "DE123465789".

  • type object

    Named type uniquely identified by its technical key

  • technicalKey string

    Unique key of this type for reference

  • name string

    Name or denotation of this type

  • issuingBody string

    The name of the official register, where the identifier is registered. For example, a Handelsregisternummer in Germany is only valid with its corresponding Handelsregister.

  • states object[]

    The list of (temporary) states of the legal entity.

  • validFrom date-time

    The date from which the state is valid.

  • validTo date-time

    The date until the state is valid.

  • type object

    Named type uniquely identified by its technical key

  • technicalKey string

    Possible values: [ACTIVE, INACTIVE]

    Unique key of this type for reference

  • name string

    Name or denotation of this type

    Name or denotation of this type

    BPN of partner which is the source of the relation

  • endBpnl string

    BPN of partner which is the target of the relation

  • validFrom date-time

    Time when the relation started

  • validTo date-time

    Time when the relation ended

  • currentness date-time

    The date the business partner data was last indicated to be still current.

  • confidenceCriteria object
  • sharedByOwner boolean
  • checkedByExternalDataSource boolean

  • lastConfidenceCheckAt date-time
  • nextConfidenceCheckAt date-time
  • confidenceLevel int32

    Indicates whether the legal entity is owned and thus provided by a Catena-X Member.

    The date when the data record has been created.

  • updatedAt date-time

    The date when the data record has been last updated.

  • legalAddress object

    In general, an address is a collection of information to describe a physical location, using a street name with a house number and/or a post office box as reference. In addition, an address consists of several postal attributes, such as country, region (state), county, township, city, district, or postal code, which help deliver mail.In Catena-X, an address is a type of business partner representing the legal address of a legal entity, and/or the main address of a site, or any additional address of a legal entity or site (such as different gates).An address is owned by a legal entity. Thus, exactly one legal entity is assigned to an address. An address can belong to a site. Thus, one or no site is assigned to an address. An address is uniquely identified by the BPNA.

  • bpna string

    A BPNA represents and uniquely identifies an address, which can be the legal address of a legal entity, and/or the main address of a site, or any additional address of a legal entity or site (such as different gates). It is important to note that only the BPNL must be used to uniquely identify a legal entity. Even in the case that the BPNA represents the legal address of the legal entity, it shall not be used to uniquely identify the legal entity.

  • name string

    The name of the address. This is not according to official registers but according to the name the sharing member chooses.

  • states object[]

    The list of (temporary) states of the address.

  • validFrom date-time

    The date from which the state is valid.

  • validTo date-time

    The date until the state is valid.

  • type object

    Named type uniquely identified by its technical key

    Possible values: [ACTIVE, INACTIVE]

    Unique key of this type for reference

  • name string

    Name or denotation of this type

  • identifiers object[]

    The value of the identifier like "0847976000005".
